### 2.2 文本问答（非流式）

**输入：**
```json
POST /ask/text
{"text":"回复OK即可","user_id":"","voice_output":false,"session_id":"","stream":false}
```
**预期：** `code=0`，返回文本回复  
**实际：** `code=0`，LLM 返回 "OK啦～"  
**结果：** ✅

### 2.3 文本问答（流式 SSE）

**输入：**
```json
POST /ask/text
{"text":"1+1","user_id":"","voice_output":false,"session_id":"","stream":true}
```
**预期：** SSE 事件流，`data:` 开头  
**实际：**
```
data: {"token": "2"}
data: {"done": true, "text": "2", "session_id": "..."}
```
**结果：** ✅

### 2.4 文本问答 + TTS 语音输出

**输入：**
```json
POST /ask/text
{"text":"说OK","user_id":"","voice_output":true,"session_id":"","stream":false}
```
**预期：** `code=0`，返回 `audio_url`  
**实际：** `code=0`，`audio_url=/ask/audio/xxx.wav`  
**结果：** ✅

## 三、语音接口测试

### 3.1 语音识别

**输入：**
```bash
POST /ask/voice
-F "audio=@data/test_audio.wav" -F "voice_output=false"
```
**预期：** ASR 识别成功，返回识别文本  
**实际：** `code=0`，`asr_text="I."`  
**结果：** ✅

### 3.2 语音识别 + TTS 合成

**输入：**
```bash
POST /ask/voice
-F "audio=@data/test_audio.wav" -F "voice_output=true"
```
**预期：** 返回 `audio_url`  
**实际：** `audio_url=/ask/audio/xxx.wav`，文件 451KB  
**结果：** ✅

### 3.3 语音流式 SSE

**输入：**
```bash
POST /ask/voice/stream
-F "audio=@data/test_audio.wav"
```
**预期：** SSE 事件 `type: asr` → `type: token` → `type: done`  
**实际：**
```
data: {"type":"asr","text":"I."}
data: {"type":"token","data":"😶"}
data: {"type":"token","data":" "}
...
```
**结果：** ✅

### 3.4 边界测试：无音源输入

**输入：**
```json
POST /ask/text {"text":"","stream":false}
```
**预期：** Pydantic 校验失败  
**实际：** 422 `"String should have at least 1 character"`  
**结果：** ✅

## 六、CRM 系统测试

### 6.1 用户创建

**输入：**
```json
POST /crm/users
{"user_id":"final-test","name":"全量测试","phone":"13800","tags":["VIP","扫地机"]}
```
**实际：** 返回完整用户信息，含创建时间  
**结果：** ✅

### 6.2 CRM 对话记录闭环

**流程：**
1. 创建用户 `crm-deep`（标签：VIP，扫地机用户）
2. 发送带 `user_id=crm-deep` 的对话："我的扫地机器人不工作了怎么办"
3. 等待 3 秒，查询对话历史

**实际对话记录：**

| 字段 | 值 |
|------|------|
| user_id | crm-deep |
| intent | function_call |
| query | 我的扫地机器人不工作了怎么办 |
| response | 先确认有没有电…检查充电座电源线… |
| created_at | 2026-06-20 09:32:31 |

**结果：** ✅ 对话正确记录，意图分类正确

### 6.3 LLM 用户画像提取

**查询用户档案：**
```json
{
  "name": "深度测试",
  "tags": ["VIP", "扫地机用户"],
  "profile": {
    "interests": ["产品故障排查"],

    "sentiment_history": [{"polarity": "neutral", "confidence": 0.6}],
    "topic_stats": {"function_call": 1}
  }
}
```

**结果：** ✅ LLM 正确提取了用户兴趣、意图和情感

### 6.4 知识库状态
